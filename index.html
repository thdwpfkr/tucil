★ Catatan Belajar

Berikut hal-hal penting yang sudah dipelajari di Web Client Development:

  1. Struktur HTML – setiap halaman web wajib punya DOCTYPE html, head, dan body.
  2. Tag formatting teks seperti bold, italic, underline, highlight, maupun strong.
  3. Perbedaan <b> dan <strong> → keduanya tebal, tapi strong punya makna semantik.
  4. Cara membuat hyperlink ke google dengan tag <a href dan menyisipkan gambar dengan <img src>.
  5. Penggunaan inline style untuk font, color, dan spacing.

Topik yang masih perlu diperdalam:

  • CSS Selector – terutama perbedaan class dan ID.
  • Box Model – memahami margin, padding, dan border secara visual.
  • Konsep specificity — aturan mana yang menang saat ada konflik style.

✎ Refleksi Pengerjaan

Approach

  • Menganalisis screenshot spesifikasi untuk memahami hierarki dokumen, penggunaan simbol Unicode/ASCII, serta struktur rowspan dan colspan pada tabel.
  • Menyusun dokumen menggunakan tag HTML semantik standar seperti header (h2, h3, h4), paragraf, baris baru, serta list (ol dan ul).
  • Mengimplementasikan elemen khusus seperti inline style untuk tag tertentu, tag formatting teks khusus, serta penyesuaian CSS border murni untuk komponen tabel saja.

Catatan

  • Bagian paling sulit adalah melakukan formatting teks pada poin Catatan Belajar nomor 5 agar persis seperti contoh, karena harus memanipulasi font, warna, dan spacing menggunakan murni inline style tanpa bantuan *external style sheet* atau kelas CSS.
  • Menentukan tata letak *rowspan* dan *colspan* pada tabel jadwal kuliah juga membutuhkan ketelitian ekstra agar struktur baris tidak berantakan saat dirender langsung di browser.
